通过事件接口配置用户的回调操作

说明： 我们可以将所有dialog的配置而都集成到现有的一个VM的对象,为了防止此对象转换为一个子VM,
我们最好让它以$ 开头,或名字放到$skipArray数组中.
然后在标签内通过config属性引用
onOpen属性定义open dialog时的回调，onConfirm定义点击确定按钮之后的回调，
onClose定义点击“close”时的回调，onCancel定义点击“取消”按钮时的回调。

                    <legend>可自定义回调操作的dialog</legend>
                    <button ms-click="show('aa')">打开对话框</button>
                    <aoyou:dialog $id="aa" config="$aaOpts" title="可自定义回调操作的dialog" >
                        呵呵呵呵呵呵呵呵呵呵呵呵呵呵呵呵呵呵呵呵呵呵呵呵呵呵
                    </aoyou:dialog>
                </fieldset>
                <script>
                    require(["./component/dialog/avalon.dialog"], function () {
                        avalon.define({
                            $id: "demo",
                            show: function (id) {
                                avalon.vmodels[id].toggle = true;
                            },
                            $aaOpts: {
                                $slot: "content", //   让<aoyou:dialog>内部的所有节点都变成组件的content属性
                                onOpen: function () {
                                    alert("open dialog");
                                },
                                onConfirm: function () {
                                    alert('success')
                                },
                                onClose: function () {
                                    alert('fail')
                                },
                                onCancel: function () {
                                    alert("cancel");
                                }
                            }
                        })
                        avalon.scan();
                    })
                </script>
            </div>
